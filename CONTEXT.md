# Project Context

## Overview

Репозиторий для написания научных статей. Каждая статья — отдельная папка с исходниками, иллюстрациями и вспомогательными материалами.

## Structure

```
articles/
├── CONTEXT.md          # этот файл — общий контекст проекта
├── <article-slug>/     # папка отдельной статьи
│   ├── draft.md        # черновик / основной текст
│   ├── references.bib  # библиография (BibTeX)
│   ├── figures/        # иллюстрации, графики
│   └── notes.md        # рабочие заметки по статье
└── ...
```

## Conventions

- Тексты пишем в Markdown (при необходимости — LaTeX для формул)
- Названия папок: латиница, kebab-case, отражают тему статьи
- Ссылки оформляем в BibTeX (`references.bib`)
- Иллюстрации в `figures/`, форматы: PNG, SVG, PDF
- Язык статей: определяется по целевому журналу/конференции

## Tech Stack

- Markdown / LaTeX
- BibTeX для библиографии
- Pandoc (опционально, для конвертации)
- Любой Markdown-редактор / IDE с AI-ассистентом

## Current Focus

### Статья 1: AI как ментор в обучении программиста

**Тезис:** AI (бесплатные LLM-модели) может выступать полноценным ментором для студентов-программистов, значительно ускоряя обучение. Метод бесплатен (при соблюдении лимитов), всегда доступен, и позволяет:
- Качать теорию (объяснения концепций в контексте проекта)
- Готовиться к собеседованиям (mock-интервью, разбор задач)
- Практиковаться (AI придумывает задачи, оценивает результат)
- Писать целые пет-проекты с поддержкой AI-ментора

**Практическая база (workspace Diploma):**

Дипломная работа описывает навчальну платформу на основі monorepo-архітектури, где AI-ассистент интегрирован как ментор. Ключевые моменты:

1. **Платформа** — monorepo (Turborepo + npm workspaces) с тремя приложениями (backend Express, frontend React, mobile React Native) и пятью shared-пакетами
2. **Методика** — спринтовая система из 4 спринтов с постепенным усложнением: авторизация → чтение данных → полный CRUD → расширенный функционал
3. **AI-ментор** — встроен в Smart IDE (Cursor, Kiro и т.д.), имеет доступ к контексту проекта (структура monorepo, стек, тикеты спринтов, архитектурные решения), работает как наставник (объясняет, направляет, но НЕ даёт готовый код)
4. **Сценарии использования AI:**
   - Объяснение теоретических понятий в контексте проекта
   - Рекомендации по реализации функционала
   - Помощь в поиске причин ошибок
   - Навигация по кодовой базе
   - Объяснение взаимодействия между модулями
5. **Результаты пилотного тестирования (5 студентов):**
   - Полезность AI: 4.8/5 (наивысший балл среди всех критериев)
   - Понятность тикетов: 4.4/5
   - Готовность к работе: 4.0/5

**Исходные файлы диплома (workspace Diploma):**
- `generate_diploma.py` — основной текст диплома (4 розділи + вступ + висновки)
- `generate_additional.py` — дополнительные подразделы (детализация)
- `EPAM_External_Lab_Tickets_*.docx` — тикеты для Backend/Frontend/Mobile команд
- `diagrams/` — диаграммы (use-case, ER, component tree, data flow и др.)
- `.mov` файлы — видеозаписи демонстрации (ask for help, code review, testing)
