# Project Context

## Overview

Репозиторий для написания научных статей. Каждая статья — отдельная папка с исходниками, иллюстрациями и вспомогательными материалами.

## Structure

```
articles/
├── CONTEXT.md          # этот файл — общий контекст проекта
├── <article-slug>/     # папка отдельной статьи
│   ├── draft.md        # черновик / основной текст
│   ├── references.bib  # библиография (BibTeX)
│   ├── figures/        # иллюстрации, графики
│   └── notes.md        # рабочие заметки по статье
└── ...
```

## Conventions

- Тексты пишем в Markdown (при необходимости — LaTeX для формул)
- Названия папок: латиница, kebab-case, отражают тему статьи
- Ссылки оформляем в BibTeX (`references.bib`)
- Иллюстрации в `figures/`, форматы: PNG, SVG, PDF
- Язык статей: определяется по целевому журналу/конференции

## Tech Stack

- Markdown / LaTeX
- BibTeX для библиографии
- Pandoc (опционально, для конвертации)
- Любой Markdown-редактор / IDE с AI-ассистентом

### Статья 1: AI як персональний ментор у підготовці фронтенд-розробників

**Файл:** `ai-mentor-frontend/draft.md`

**Статус:** перший драфт готовий, на ревʼю

**Назва:** «Штучний інтелект як персональний ментор у процесі підготовки фронтенд-розробників у закладах вищої освіти»

**Мова:** українська

**Формат:** наукова стаття для університетського збірника

**Тезис:** Безкоштовні LLM-моделі можуть ефективно виконувати роль ментора для студентів-програмістів: пояснюють теорію (готові пояснювати 100 разів), генерують задачі, аналізують код, готують до співбесід, менторять пет-проекти. Метод безкоштовний, доступний 24/7, персоналізований через контекст проекту.

**Структура статті (7 розділів):**
1. Вступ — проблема, мета, об'єкт/предмет/методи
2. Проблема підготовки програмістів у ЗВО — розрив теорія/практика, обмеження існуючих форматів
3. AI як ментор: теоретичне обґрунтування — можливості LLM, переваги, обмеження (з контекстом що ціна помилки мала), вимоги до студента
4. Методика навчання — цикл теорія→практика→рефлексія, AI генерує проект/задачі, підготовка до співбесід, пет-проекти
5. Приклад практичної реалізації — спринтова платформа, контекстуалізація у Smart IDE
6. Експериментальна апробація — 5 студентів, корисність AI 4.8/5
7. Висновки та перспективи

**Ілюстративний матеріал (таблиці):**
- Табл. 1 — Результати пілотного тестування
- Табл. 2 — Циклічна модель навчання
- Табл. 3 — Порівняння підходів (лекції / курси / ментор / AI)
- Табл. 4 — Схема взаємодії у Smart IDE
- Табл. 5 — Спринтова прогресія

**Джерела:** 10 джерел (Stack Overflow 2025, Nature 2025, arXiv 2025, Frontiers 2026, Brookings 2025 та ін.)

**Зв'язок з дипломом (workspace Diploma):**
- Диплом використано як джерело практичного досвіду та даних експерименту
- Статья НЕ крутиться навколо диплому — він лише приклад реалізації (розділ 5) та джерело даних (розділ 6)
- Фокус статті — методика AI-менторства в цілому

**Принципові рішення:**
- Без згадування конкретних платформ/брендів (Udemy, Coursera і т.д.)
- Без точних цін за менторство — просто "фінансово затратне"
- AI сам генерує проект і задачі для студента (студент обирає домен)
- Слово "тікети" замінено на "задачі"
- Назви інструментів (Cursor, Kiro) — абстрактно "Smart IDE"

### Статья 2: AI як інструмент для викладачів IT-дисциплін

**Файл:** `ai-for-it-teachers/draft.md`

**Статус:** перший драфт готовий, на ревʼю

**Назва:** «Використання штучного інтелекту як інструменту підвищення ефективності роботи викладачів IT-дисциплін у закладах вищої освіти»

**Мова:** українська

**Формат:** наукова стаття для університетського збірника

**Тезис:** AI автоматизує рутину викладача (перевірка, генерація завдань, актуалізація курсу), що трансформує його роль — від шаблонного лектора-бюрократа до ментора-креатора, який фокусується на якості результату (підготовлений студент) та має простір для креативності.

**Структура статті (8 розділів):**
1. Вступ — проблема, мета, об'єкт/предмет/методи
2. Проблеми викладання IT-дисциплін — старіння контенту, рутина, вигорання
3. Можливості AI — актуалізація, генерація завдань, перевірка, матеріали, зворотний зв'язок
4. Методика інтеграції — принцип «AI чернетка → викладач рішення»
5. Трансформація ролі викладача — від лектора до ментора (ключовий розділ!)
6. Практичні сценарії — 5 конкретних прикладів з оцінкою економії часу
7. Обмеження та рекомендації — верифікація, етика, компетенції, поетапне впровадження
8. Висновки

**Ілюстративний матеріал (таблиці):**
- Табл. 1 — Розподіл відповідальності AI / викладач
- Табл. 2 — Трансформація ролі (було → стало)
- Табл. 3 — Сценарії та економія часу (без AI vs з AI)
- Табл. 4 — Поетапне впровадження

**Джерела:** 10 джерел (Stack Overflow 2025, arXiv 2025, Nature 2025, Frontiers 2026, Brookings 2025, Stanford 2025)
